## 一、产品概述

### 1.1 产品定位

一款专为大学生设计的课程表应用，支持小米澎湃OS超级岛（Live Activities）功能，让用户在不打开应用的情况下即可查看当前课程信息。

### 1.2 目标用户

- 在校大学生
- 研究生
- 需要管理课程时间的教育工作者

### 1.3 核心价值

- **便捷性**：一目了然的课程表视图
- **实时性**：通过小米超级岛实时显示当前课程
- **个性化**：支持自定义课程颜色和时间设置

## 二、功能需求

### 2.1 课程管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 添加课程 | 输入课程名称、教师、地点、时间等信息 | P0 |
| 编辑课程 | 修改已有课程的所有信息 | P0 |
| 删除课程 | 删除不需要的课程 | P0 |
| 课程颜色 | 为不同课程设置不同颜色标识 | P1 |

### 2.2 课程表展示

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 周次切换 | 支持1-20周的课程查看 | P0 |
| 日视图 | 按天展示课程列表 | P1 |
| 周视图 | 网格形式展示整周课程 | P0 |
| 当前周高亮 | 标记当前正在进行的课程 | P1 |

### 2.3 小米超级岛

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 实时显示 | 显示当前正在进行的课程 | P0 |
| 下一节课 | 预告下一节课信息 | P0 |
| 自动更新 | 课程结束后自动切换显示 | P0 |
| 点击跳转 | 点击通知打开应用 | P1 |

### 2.4 数据管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 本地存储 | 使用SharedPreferences持久化数据 | P0 |
| 数据备份 | 导出课程数据为JSON | P2 |
| 数据恢复 | 从JSON导入课程数据 | P2 |

## 三、用户场景

### 场景一：添加新学期课程

```
用户：小明，大三学生
时间：新学期开学前
操作：
1. 打开应用
2. 点击右上角"+"按钮
3. 输入"高等数学"课程信息
4. 设置周一 1-2节，张教授，教301
5. 设置第1-16周
6. 保存课程
```

### 场景二：上课时查看课程信息

```
用户：小红，正在去教室的路上
时间：周一上午 7:55
操作：
1. 亮屏手机
2. 在小米超级岛上看到：
   - 当前课程：高等数学
   - 教室：教301
   - 时间：8:00-9:40
   - 下一节：大学英语 10:00
3. 直接前往教室，无需打开应用
```

### 场景三：查看本周课程安排

```
用户：小刚，周日晚上规划下周学习
时间：周日 21:00
操作：
1. 打开应用
2. 查看本周课程表网格
3. 切换到下周查看课程变化
4. 根据课程安排制定学习计划
```

## 五、数据模型

### 5.1 课程数据结构

```json
{
  "id": "uuid-string",
  "name": "高等数学",
  "teacher": "张教授",
  "location": "教301",
  "dayOfWeek": 1,
  "startSection": 1,
  "endSection": 2,
  "startTime": "08:00",
  "endTime": "09:40",
  "color": "#2196F3",
  "startWeek": 1,
  "endWeek": 16,
  "isOddWeek": false,
  "isEvenWeek": false
}
```

### 5.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 唯一标识符 (UUID) |
| name | String | 课程名称 |
| teacher | String | 授课教师 |
| location | String | 上课地点 |
| dayOfWeek | int | 星期几 (1-7) |
| startSection | int | 开始节次 (1-10) |
| endSection | int | 结束节次 (1-10) |
| startTime | String | 开始时间 (HH:mm) |
| endTime | String | 结束时间 (HH:mm) |
| color | String | 颜色代码 (#RRGGBB) |
| startWeek | int | 开始周次 |
| endWeek | int | 结束周次 |
| isOddWeek | bool | 是否单周 |
| isEvenWeek | bool | 是否双周 |

## 七、权限说明

### 7.1 Android权限

| 权限 | 用途 | 必要性 |
|------|------|--------|
| POST_NOTIFICATIONS | 发送通知 (Android 13+) | 必需 |
| VIBRATE | 通知振动 | 可选 |
| FOREGROUND_SERVICE | 前台服务 | 可选 |

### 7.2 权限申请时机

1. **首次启动**：申请通知权限
2. **首次显示超级岛**：确保通知渠道创建成功
